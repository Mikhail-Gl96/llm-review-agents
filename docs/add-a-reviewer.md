# Как добавить нового ревьюера

Тулкит устроен так, чтобы ревьюеров было легко наращивать. Каждый ревьюер — **отдельная
самодостаточная папка** `reviewers/<name>/`; добавление нового **не трогает** существующих.

## Принцип

Единый шаблон папки (образец — [`reviewers/pr-agent/`](../reviewers/pr-agent/README.md)):

```
reviewers/<name>/
  README.md        # обзор ревьюера + навигация (что делает, кому, ссылки)
  local.md         # локально: установка, конфиг провайдера, запуск
  ci.md            # интеграция в CI (какие секреты, ссылки на ci/*)
  setup/           # скрипты установки и/или шаблоны конфигов
  ci/              # готовые файлы под платформы (github-actions.yml, gitlab-ci.yml, ...)
```

## Шаги

1. **Создай папку** `reviewers/<name>/` с этой структурой (проще всего — скопировать
   `reviewers/pr-agent/` и переписать под свой инструмент).
2. **Заполни `README.md`** — 3-4 строки «что это / кому» + навигация на `local.md`, `ci.md`, `setup/`.
3. **`local.md`** — установка и запуск. Провайдеров **не дублируй**: ссылайся на
   [`docs/providers.md`](providers.md) (из файла ревьюера путь — `../../docs/providers.md`),
   показывай только маппинг провайдера в конфиг своего инструмента.
4. **`ci.md` + `ci/`** — как встроить в пайплайн и готовые файлы под платформы.
5. **Добавь строку в таблицу-навигатор** корневого [`README.md`](../README.md): что делает /
   стек / кому / ссылки «Локально» · «CI» (· «По комментарию», если поддержан ботом).

## Чек-лист готовности

- [ ] Файлы на месте: `README.md`, `local.md`, `ci.md`, `setup/`, `ci/`.
- [ ] Скрипты проходят `bash -n` (и `shellcheck`, если установлен).
- [ ] CI-файлы — валидный YAML (`ruby -ryaml -e 'YAML.load_file(ARGV[0])' <файл>`).
- [ ] Относительные ссылки резолвятся (линк-чек из корневого README не находит битых).
- [ ] Строка про новый ревьюер добавлена в таблицу корневого `README.md`.

## Чтобы ревьюер вызывался ботом (по комментарию)

Режим «тегнуть бота в PR → тогда ревью» — это модуль `bot/` (подпроект SP2). Чтобы новый
ревьюер вызывался по имени (напр. `@review.<name>`), в боте добавляется адаптер
(`run(checkout, base, head) → markdown`) и строка в реестре. Подробности — в `bot/README.md`
(когда модуль появится).
